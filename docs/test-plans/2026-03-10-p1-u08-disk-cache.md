## Human Verification Required

| Concern                              | Why Manual                                                                                                         | Steps                                                                                                                                                                                                                                                             |
| ------------------------------------ | ------------------------------------------------------------------------------------------------------------------ | ----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Atomic write correctness under crash | Automated test (AC2.3) checks no `.tmp` files remain after successful flush, but cannot simulate a mid-write crash | Review `flush()` in `disk-cache.ts`. Confirm: (1) data files are fsynced before index write, (2) index is written to `.tmp` first, (3) `fsync` called on `.tmp`, (4) `rename()` atomically replaces `index.json`. This is the standard POSIX atomic-write pattern |
| Concurrent flush safety              | No tests for concurrent `flush()` calls; no locking implemented                                                    | Review `flush()` to confirm two concurrent calls could interleave writes. If single-threaded sync engine is the only caller, this is acceptable. If not, file an issue                                                                                            |
| Factory counter leakage              | `makeRecipe()` / `makeCategory()` use module-level counters that persist across tests in the same vitest worker    | Run full test suite and confirm all tests pass. Tests needing specific UIDs pass explicit `uid` overrides, mitigating flakiness risk                                                                                                                              |
