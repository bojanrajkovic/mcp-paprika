# Test Requirements: discover_recipes Tool

## Automated Tests

All acceptance criteria are verifiable through unit tests using the established tool testing pattern (`makeTestServer()` + `makeCtx()` + `getText()`) with a mocked `VectorStore`.

| AC ID | Description                                                                                         | Test Type | Test File                    | What to Verify                                                                                                                                                                                                                 |
| ----- | --------------------------------------------------------------------------------------------------- | --------- | ---------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ |
| AC1.1 | Tool is registered with name `discover_recipes`                                                     | Unit      | `src/tools/discover.test.ts` | Register the tool via `registerDiscoverTool`, invoke it by name through the test server -- should execute without a "tool not registered" error.                                                                               |
| AC1.2 | `query` parameter is required (string)                                                              | Unit      | `src/tools/discover.test.ts` | Call with a `query` string and confirm the handler executes successfully. Implicitly verified by every other test that invokes the tool with a query.                                                                          |
| AC1.3 | `topK` parameter is optional, integer, 1-20, defaults to 5                                          | Unit      | `src/tools/discover.test.ts` | (a) Call without `topK` and assert `vectorStore.search` was called with `topK = 5`. (b) Call with `topK = 10` and assert `vectorStore.search` was called with `10`.                                                            |
| AC2.1 | `vectorStore.search(query, topK)` is called with both parameters from input                         | Unit      | `src/tools/discover.test.ts` | Mock `vectorStore.search`, call the tool with specific `query` and `topK` values, assert the mock was called with those exact arguments.                                                                                       |
| AC2.2 | Each result includes recipe name with similarity as integer percentage                              | Unit      | `src/tools/discover.test.ts` | Mock returns `[{ uid, score: 0.923, recipeName }]` with a matching recipe in the store. Assert output contains the recipe name and `92% match`.                                                                                |
| AC2.3 | Categories resolved via `ctx.store.resolveCategories()` and displayed when present                  | Unit      | `src/tools/discover.test.ts` | Load a recipe with categories into the store, mock returns the matching UID. Assert output contains the resolved category names. Also test a recipe with no categories and confirm the categories line is absent.              |
| AC2.4 | `prepTime` and `cookTime` displayed when present, omitted when null                                 | Unit      | `src/tools/discover.test.ts` | (a) Recipe with `prepTime: "10 min"` and `cookTime: "30 min"` -- assert output contains both time strings. (b) Recipe with `prepTime: null, cookTime: null` -- assert output contains neither time string.                     |
| AC2.5 | Each result includes `UID: \`{uid}\``                                                               | Unit      | `src/tools/discover.test.ts` | Assert the output text contains `` UID: `<expected-uid>` `` for each result in backtick-wrapped inline code format.                                                                                                            |
| AC3.1 | `search()` returns empty array -- tool returns no-results message                                   | Unit      | `src/tools/discover.test.ts` | Mock `search` to return `[]`. Assert output is exactly `"No recipes found matching that description."`                                                                                                                         |
| AC3.2 | All results map to deleted recipes -- tool returns no-results message                               | Unit      | `src/tools/discover.test.ts` | Mock `search` to return results whose UIDs do not exist in the `RecipeStore`. Assert output is exactly `"No recipes found matching that description."`                                                                         |
| AC4.1 | Results where `ctx.store.get(uid)` returns `undefined` are silently skipped                         | Unit      | `src/tools/discover.test.ts` | Mock returns 3 results; only 2 UIDs exist in the store. Assert output contains exactly 2 formatted results and no mention of the deleted recipe.                                                                               |
| AC4.2 | Remaining results are re-numbered sequentially (no gaps)                                            | Unit      | `src/tools/discover.test.ts` | Same setup as AC4.1. Assert the output numbers are `1.` and `2.` with no gap (e.g., no `3.` or skipped `2.`).                                                                                                                  |
| AC5.1 | When `ctx.store.size === 0`, tool returns cold-start message without calling `vectorStore.search()` | Unit      | `src/tools/discover.test.ts` | Create an empty `RecipeStore` (no recipes loaded). Call the tool and assert: (a) output contains the cold-start / "try again" message, (b) `vectorStore.search` was NOT called (`expect(mock.search).not.toHaveBeenCalled()`). |

## Human Verification

None -- all criteria are automatable.
